Generic REST JPA

How about a Library, that used on a JPA Entity, generates a complete REST solution using CRUD and writes the unit tests for you ?

Generic REST JPA is a JEE library targeted on creating generic Resource Services and Data Access Layers on top of JPA and CDI.

It develops REST Services starting from the CRUD pattern and extends it by adding frequently needed methods to each domain entity.

The idea behind it is that just by adding these generic services on each entity of your database you get a general 70 - 90% implementation of the services needed by the application, thus allowing the developer to focus just on the complex cases, in other words removing as much as "boilerplate code" as possible.

And that comes with already developed test units that bring 100% coverage to all provided service methods.

Interested ? Have a look at the Generic REST JPA repository on GitHub.

Annotator

You have to implement a solution that uses many annotations, and you already had a lot of issues handling them ?

Annotator is a plain Java library targeted on delivering an uniform reflection and annotation API over multiple ORM use cases.

The API is defined in the main Interface Annotate. Given a specific scope that can be a class/instance/field/method/accessors, 14 methods are available.

Using the annotator class through this API multiple points of interest can be queried from the ORM perspective : classes, fields, methods, objects.
In other words, learn once - use everywhere.

Check out the Annotator repository on GitHub.
